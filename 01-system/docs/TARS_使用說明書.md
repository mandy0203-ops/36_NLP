**版本**: 1.0  
**更新日期**: 2025-11-26

## 1. 系統簡介 (Introduction)

我是 **TARS-001**，您的專屬 AI 協作代理人 (Agentic Coding Assistant)。我的設計目標是協助您處理程式開發、系統運維及媒體處理任務。

### 核心原則 (Prime Directives)
1.  **繁體中文優先**：所有的溝通、報告及生成的檔案名稱（在技術可行下）皆使用繁體中文。
2.  **本地優先 (Local-First)**：所有操作優先在您的本機執行，確保資料隱私與速度。
3.  **安全至上**：絕不洩漏 API Key，並嚴格遵守檔案操作權限。
4.  **非破壞性**：未經確認，絕不刪除或覆蓋您的重要資料。

---

## 2. 檔案結構 (File Structure)

為了保持工作區整潔，請遵守以下目錄規範：

- **`02-inputs/` (輸入區)**：
    - 請將所有需要我處理的原始檔案（如影片、音檔、文件）放入此處。
    - **注意**：此區對我來說是「唯讀」的，我不會修改這裡的原始檔。

- **`03-outputs/` (輸出區)**：
    - 所有我生成的成品（如字幕檔、剪輯影片、報告）都會放在這裡。
    - 檔案會依照「工具名稱」與「時間戳記」自動分類，例如：`03-outputs/audio_transcribe/20251126_會議記錄/`。

- **`01-system/` (系統核心)**：
    - 我的大腦與工具箱。除非您清楚自己在做什麼，否則請勿隨意修改此目錄下的內容。

## 3. 常用指令與劇本 (Playbooks)

您可以直接使用自然語言對我下達以下指令，我會自動對應到內建的標準作業流程：

### 🎥 媒體下載
*   **指令**：「幫我下載這個 YouTube 影片」
*   **功能**：使用 `yt-dlp` 下載最高畫質影片（預設 1080p MP4）。
*   **輸出**：`03-outputs/youtube_downloader/downloads/`

### 📝 語音轉文字 (STT)
*   **指令**：「轉錄這個音檔」、「幫我把這影片轉成字幕」
*   **功能**：
    - 支援 **Groq (Whisper)** 與 **ElevenLabs** 雙引擎。
    - 自動進行「智慧分段」、「標點符號修正」與「繁體中文優化」。
    - 支援自訂詞典（修正專有名詞）。
*   **輸出**：包含 `.srt` (字幕) 與 `.txt` (逐字稿) 的資料夾。

### 🎞️ YouTube 一鍵轉字幕
*   **指令**：「YouTube 轉字幕 [URL]」
*   **功能**：自動下載影片音軌 -> 轉錄 -> 生成字幕 -> 清理暫存檔。適合快速獲取影片內容。

### 📂 專案級轉錄
*   **指令**：「幫我轉錄這個資料夾裡的所有錄音」
*   **功能**：批次處理資料夾內所有音檔，並彙整成一份 Markdown 總表。

## 5. 互動建議 (Best Practices)

為了讓我們合作更順暢：

1.  **指令明確**：盡量包含具體的「動作」與「目標檔案」。
    *   ❌ 「處理一下這個」
    *   ✅ 「請用 Groq 引擎轉錄 `02-inputs/meeting.mp3`」
2.  **檢查路徑**：提到檔案時，若能確認檔案已在 `02-inputs/` 中，會節省很多確認時間。
3.  **反饋修正**：如果生成的結果不滿意（例如字幕時間軸不準），請直接告訴我，我會調整參數重試。
